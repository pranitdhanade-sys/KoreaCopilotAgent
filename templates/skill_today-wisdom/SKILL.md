# 오늘의 명언 (today-wisdom)

## Overview

사용자가 하루를 시작하거나 마무리할 때, 또는 동기부여가 필요할 때
의미 있는 명언 하나를 골라 전달한다. 한국어 번역과 함께 원문(영어 등),
저자 이름, 그리고 그 말이 왜 울림을 주는지에 대한 짧은 해설을 함께 제시한다.

매번 다른 주제와 분위기의 명언을 제공해 진부하지 않도록 한다.

## When to Use

다음과 같은 말을 들었을 때 이 스킬을 실행한다:
- "오늘의 명언"
- "명언 하나"
- "명언 하나 알려줘"
- "동기부여 해줘"
- "오늘의 한마디"
- "좋은 말 한마디"

## When NOT to Use

- 사용자가 특정 인물의 명언만 모아달라고 하는 경우 → 일반 검색으로 처리
- 사용자가 책 전체 요약·인용집을 요청하는 경우 → 별도 리서치 작업
- 사용자가 자기 글에 인용할 출처가 검증된 문구를 요구하는 경우
  → `deep-research` 에이전트로 라우팅 (출처 확인 필요)

## Quick Start

1. 진행 상황 작업 하나 생성 ("오늘의 명언 고르기")
2. 주제 풀에서 오늘의 테마를 무작위로 선택
3. 해당 테마에 맞는 명언 하나를 고름
4. 형식에 맞춰 출력 (한국어 + 원문 + 저자 + 짧은 해설)
5. 작업 완료 처리

## Core Instructions

### Progress Tracking

요청을 받으면 즉시 `TaskCreate`로 "오늘의 명언 고르기" 작업 하나를 만든다.
출력 직전에 `TaskUpdate`로 완료 처리한다. 한 번에 끝나는 작업이므로
세분화하지 않는다.

### Phase 1 — 테마 선택

매번 다른 분위기의 명언을 제공하기 위해 다음 테마 풀 중 하나를 골라 사용한다:

| 테마 | 분위기 |
|------|--------|
| 동기부여 | 행동을 부르는 강한 메시지 |
| 지혜·성찰 | 깊이 생각하게 하는 통찰 |
| 일과 성공 | 커리어·리더십·성장 |
| 도전과 용기 | 두려움을 마주하는 힘 |
| 관계·사랑 | 사람 사이의 따뜻함 |
| 일상의 감사 | 작은 것에서 발견하는 행복 |
| 시간과 변화 | 흐름과 선택에 대한 통찰 |

같은 테마가 연속해서 나오지 않도록 다양성을 의식적으로 유지한다.

### Phase 2 — 명언 선정

선정 원칙:
- 실존 인물의 실제 발언만 사용 (출처가 불분명한 문구는 피한다)
- 한 줄로 정리될 수 있는 간결한 명언 우선
- 너무 진부한 문구(예: "노력은 배신하지 않는다") 보다는
  덜 알려졌지만 울림이 있는 문구 선호
- 저자가 영어권이 아니면 가능한 원어를 함께 제시, 어려우면 영어 번역 사용

**중요:** 저자나 출처가 확실하지 않으면 해당 명언은 사용하지 않는다.
"작자 미상"이나 "어느 철학자가 말하길" 같은 표현은 쓰지 않는다.

### Phase 3 — 출력 형식

다음 형식을 정확히 따른다:

```
🌿 오늘의 명언

"[한국어 번역]"

[원문 — 영어 또는 원어]

— [저자 이름]

💭 [2~3문장의 짧은 해설: 이 말이 어떤 상황에서 울림을 주는지,
   오늘 하루에 어떻게 적용해 볼 수 있는지]
```

이모지는 위 두 개(🌿, 💭)만 사용한다. 다른 장식은 추가하지 않는다.

### Available MCP Tools

이 스킬은 외부 도구를 거의 사용하지 않는다. 명언 자체는 모델 내부 지식에서
가져온다. 다만 다음 경우에 한해 도구 사용을 고려한다:

| 상황 | 도구 |
|------|------|
| 사용자가 출처 검증을 요청한 경우 | `web_search` |
| 사용자가 최근 인물(현직 CEO·작가 등)의 발언을 원할 때 | `web_search` |

기본 흐름에서는 어떤 MCP 도구도 호출하지 않는다.

## Examples

### Example 1 — 아침 동기부여

**사용자:** "오늘의 명언"

**출력:**
```
🌿 오늘의 명언

"움직이고 있는 한, 속도는 중요하지 않다."

"It does not matter how slowly you go as long as you do not stop."

— 공자 (Confucius)

💭 오늘 해야 할 일이 너무 크게 느껴질 때 떠올리기 좋은 말이에요.
   완벽한 속도보다 멈추지 않는 꾸준함이 결국 더 멀리 데려다 줍니다.
```

### Example 2 — 일과 리더십 테마

**사용자:** "동기부여 해줘"

**출력:**
```
🌿 오늘의 명언

"리더가 된다는 것은, 다른 사람을 더 나은 사람으로 만드는 일이다."

"Leadership is about making others better as a result of your presence
and making sure that impact lasts in your absence."

— 셰릴 샌드버그 (Sheryl Sandberg)

💭 오늘 회의나 동료와의 대화에서, 내 말이 상대를 한 뼘 더 자라게
   하는지 가만히 들여다 보세요. 리더십은 직급이 아니라 영향력입니다.
```

### Example 3 — 성찰 테마

**사용자:** "오늘의 한마디"

**출력:**
```
🌿 오늘의 명언

"우리는 우리가 반복해서 하는 행동의 결과이다.
탁월함은 행위가 아니라 습관이다."

"We are what we repeatedly do. Excellence, then, is not an act,
but a habit."

— 아리스토텔레스 (Aristotle)

💭 오늘의 작은 선택들이 곧 내일의 나를 만듭니다. 거창한 결심보다,
   지금 이 한 시간을 어떻게 보낼지 결정하는 것이 더 큰 변화를 만듭니다.
```

## Guardrails

1. **출처가 확실한 명언만 사용** — 작자 미상이나 인터넷 떠도는 문구는 피한다.
   확신이 없으면 다른 명언을 고른다.
2. **저자 이름 조작 금지** — 실제로 말하지 않은 사람에게 명언을 귀속시키지 않는다.
3. **종교·정치적으로 편향된 명언은 피한다** — 보편적 공감을 줄 수 있는 메시지 우선.
4. **사용자 상황을 단정하지 않는다** — "힘드시죠?" "지치셨죠?" 같은 추측 표현 금지.
   해설은 명언 자체의 의미에 집중한다.
5. **연속 호출 시 다양성 유지** — 같은 저자·같은 테마가 연달아 나오지 않도록 한다.
6. **번역 품질** — 직역체보다 자연스러운 한국어. 원문의 운율과 의미가
   잘 살도록 의역을 허용한다.

## Common Issues

| 문제 | 처리 방법 |
|------|----------|
| 출처를 확신할 수 없는 명언이 떠오름 | 사용하지 않고 다른 명언을 고른다 |
| 사용자가 "다른 거" 요청 | 다른 테마에서 새로운 명언 선정 |
| 사용자가 특정 인물 요청 (예: "스티브 잡스 명언") | 형식은 유지하되 해당 인물의 검증된 명언 사용 |
| 사용자가 원문 없이 보고 싶다고 함 | 원문 줄을 제외하고 한국어와 저자만 출력 |
| 너무 잘 알려진 진부한 문구만 떠오름 | 덜 알려진 작가나 다른 테마로 전환 |
| 사용자가 출처를 검증해 달라고 함 | `web_search`로 확인 후 결과 보고 |
| 한국어 번역이 어색함 | 의역으로 자연스러움 우선, 원문은 그대로 유지 |
